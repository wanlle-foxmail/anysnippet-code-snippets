# Nginx Basic Auth Example

This snippet uses a Markdown guide with a Mermaid diagram and the official Nginx basic auth directives to explain the smallest protected-location setup.

## What This Snippet Covers

- One protected `location /` block
- Browser login prompt via `auth_basic`
- Password file lookup via `auth_basic_user_file`

## Official Correctness Note

This snippet follows the official Nginx `ngx_http_auth_basic_module` documentation:

- https://nginx.org/en/docs/http/ngx_http_auth_basic_module.html

The reload note in this snippet also matches the Nginx beginner's guide:

- https://nginx.org/en/docs/beginners_guide.html

## Entry File

- `nginx-basic-auth.md`
- The entry file intentionally stays in Markdown so the request flow and the config fragment remain together on the first screen.

## How to Use It

- Open `nginx-basic-auth.md` first.
- Paste the `location /` block into an existing `server {}` block.
- Create the password file referenced by `auth_basic_user_file` with the `openssl passwd -apr1` example shown in `nginx-basic-auth.md`.
- Run `nginx -t`, then reload Nginx.

## Manual Verification

1. Open `nginx-basic-auth.md` and confirm the request-flow diagram matches your intended behavior.
2. Create an htpasswd-compatible password file at the path used by `auth_basic_user_file`, for example with `openssl passwd -apr1` as shown in `nginx-basic-auth.md`.
3. Paste the config block into a staging Nginx server configuration.
4. Run `nginx -t` on the staging host.
5. Reload Nginx.
6. Request the protected path and confirm Nginx returns a browser auth prompt.
7. Retry with valid credentials and confirm the protected location becomes accessible.

## Notes

- The official module docs allow these directives in `http`, `server`, `location`, and `limit_except`; this snippet chooses `location /` because it is the simplest protected-path example.
- The official docs mention password files generated by Apache `htpasswd` or `openssl passwd`.
